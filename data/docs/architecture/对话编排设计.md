# 对话编排设计

**上游**：[技术方案设计 §3.2](技术方案设计.md) · [66题维度映射](../product/66题维度映射.md)

## 总体原则

用户感知层是"和 AI 朋友打电话"；系统编排层是"有目标、可控、可追踪的人格采样引擎"。两者通过下方状态机和双 LLM 架构耦合。

## 会话状态机

```
                      ┌───────────────────────────┐
                      │      SESSION FSM           │
                      │  (per-user, per-session)   │
                      └──────────┬────────────────┘
                                 �� user starts call
                                 ▼
                         ┌──────────────┐
                         │   OPENING    │  AI 主动开场白（≤30s）
                         └──────┬───────┘
                                │
                                ▼
                         ┌──────────────┐
              ┌─────────►│  CHATTING    │◄──────────┐
              │          │  (核心阶段)   │           │
              │          └──────┬───────┘           │
              │                 │                    │
              │     ┌───────────┴───────────┐       │
              │     ▼                       ▼       │
        ┌───────────────┐          ┌────────────────┐
        │  PROBING       │          │  COMFORTING    │
        │  (维度采样轮)   │          │  (情绪安抚轮)  │
        └───────┬───────┘          └────────┬───────┘
                │                           │
                └───────────┬───────────────┘
                            │
                ┌───────────┴───────────┐
                │   coverage ≥ target   │  OR  duration ≥ 13min
                │   OR user says bye    │
                ▼                       │
         ┌──────────────┐              │
         │  CLOSING      │◄────────────┘
         │  (自然收尾)    │
         └──────┬───────┘
                │ AI 说结束语
                ▼
         ┌──────────────┐
         │   ENDED       │  写入画像、标记已覆盖维度
         └──────────────┘
```

### 状态说明

| 状态 | 进入条件 | 退出条件 | 行为 |
|---|---|---|---|
| OPENING | 用户接通 | 用户首句到达 | AI 说"嗨，今天过得怎么样？"等开场白 |
| CHATTING | 开场结束 | 编排层判断需采样或安抚 | 自由聊天，AI 随意跟随话题 |
| PROBING | 维度未覆盖 | 获得一条有效证据 | 编排层注入 `probe_hint`，LLM 自然引导 |
| COMFORTING | 检测到负面情绪（≥阈值）| 情绪回正 | 暂停采样，切共情安抚 |
| CLOSING | 覆盖率达标 / 时长 ≥ 13min / 用户主动结束 | AI 结束语后 | 自然过渡结束 |
| ENDED | AI 收尾完毕 | — | 写库 + 异步画像抽取 |

### 时长约束

- `duration < 5min`：不触发 CLOSING（产品要求最少 5 分钟）
- `duration ≥ 13min`：强制进入 CLOSING（留 2 分钟收尾，总上限 15min）
- 用户发送 `extend`：延长至当前时间 +5min，最长 20min

## 双 LLM 架构

```
用户这一轮说的话（ASR 文本）
    │
    ▼
┌────────────────────────────────┐
│  LLM #1 — 理解 + 抽取         │
│  model: 小模型 (豆包lite等)    │
│  input:                        │
│    - system: 你是证据抽取器    │
│    - context: 最近 5 轮对话    │
│    - user_msg: ASR 文本        │
│  output (JSON):                │
│    { intent, emotion,          │
│      evidence: [{dim, text,    │
│        confidence_delta}],     │
│      covered_questions: [Qxx] }│
└───────────┬────────────────────┘
            │
            ▼
┌────────────────────────────────┐
│  规则引擎 — 状态更新           │
│  1. 合并 evidence → profile    │
│  2. 更新 covered_questions     │
│  3. 计算 dimension confidence  │
│  4. 选择下一步策略:            │
│     a. 情绪 < 阈值? → COMFORT │
│     b. 需采样? → 选维度 →     │
│        取 probe_hint           │
│     c. 覆盖饱和? → CLOSING    │
│     d. else → free CHATTING   │
│  5. 输出: next_state,          │
│     probe_hint?, constraint    │
└───────────┬────────────────────┘
            │
            ▼
┌────────────────────────────────┐
│  LLM #2 — 生成回复             │
│  model: 大模型 (豆包pro等)     │
│  input:                        │
│    - system: 人设 + 阶段约束   │
│    - long_term_memory: top-K   │
│    - context: 最近 10 轮       │
│    - [if PROBING]:             │
│        "自然引向: {probe_hint}"│
│    - [if COMFORTING]:          │
│        "先共情，不要追问"      │
│    - [if CLOSING]:             │
│        "自然收尾，总结这次聊天"│
│  output: AI 回复文本           │
└───────────┬────────────────────┘
            │
            ▼
          TTS → 下发语音
```

## 维度采样优先级算法

每轮 PROBING 选哪个维度：

```
score(dim) = weight(dim)
           × (1 - confidence(dim))
           × time_decay(last_probed)
           × (1 + emotion_compatibility(dim, current_emotion))
```

- `weight`：维度匹配权重（高 / 中 / 低）
- `1 - confidence`：置信度越低越优先
- `time_decay`：上一次触及该维度距今越久越优先
- `emotion_compatibility`：情绪低时不探 D7（恐惧）、D10（社会态度），优先 D4/D8（轻松话题）

取 `score` 最高的维度 → 从 `questions` 表随机取一条 `probe_hint`（未覆盖的优先）。

## 会话结束后处理

1. **同步**：`voice_sessions` 状态 → `ended`，写 `duration_sec`
2. **异步 worker**：
   - 汇总本次全部 evidence → 更新 `profile_documents` JSONB
   - 更新 `covered_questions`
   - 如果本次新覆盖了某个维度的首条证据 → 生成 insight 文案 → 写 `memory/insights`
   - 为证据不足维度生成 `next_probe_hints`，存入 Redis，下轮开场用

## 多轮渐进策略

| 轮次 | 目标 | 预期覆盖 |
|---|---|---|
| 第 1 轮（首次通话 15min） | 建立信任 + 基础摸底 | D4/D5/D6 + D8 部分 |
| 第 2 轮 | 深入价值观和关系 | D1/D3/D7 |
| 第 3 轮 | 补全未覆盖 + 开放题 | D2/D9/D10 + Q66 |
| 第 4 轮起 | 追踪变化 + 持续校正 | 低置信度维度复核 |

## 安全边界

- 用户提到自杀 / 自残 / 暴力相关 → 立即切 COMFORTING + 输出安全话术 + 后台告警
- 用户要求"别问了" → 停止当前维度采样，标记为 user_declined，不再主动触及该维度
- 敏感维度（D10 社会态度） → 只在用户主动提及时被动采样，不主动引导
