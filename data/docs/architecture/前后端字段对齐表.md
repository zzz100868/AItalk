# 前后端字段对齐表

**原则**：后端 API 返回的字段名、类型、结构严格贴合前端 `data` 中的字段。前端不改，后端做 DTO 映射。

## 1. 聊天页（pages/index/index）

前端 `data`：

| 字段 | 类型 | 含义 | 后端来源 |
|---|---|---|---|
| `isCalling` | bool | 当前是否通话中 | 前端本地状态 + WS 连接状态 |
| `callDuration` | string "mm:ss" | 通话时长 | 前端本地计时（后端也记，但前端自行显示） |
| `callDate` | string | 上次通话时间 | `GET /api/voice/sessions?limit=1` → `ended_at` 格式化为 "今天 HH:mm" |
| `isMuted` | bool | 静音 | 前端本地 |
| `isSpeakerOn` | bool | 免提 | 前端本地 |
| `aiName` | string | AI 名字 | 固定 "小雅"，或 `GET /api/me` → `ai_friend_name` |
| `aiAvatar` | string | AI 头像 URL | 固定值或后端配置 |
| `userAvatar` | string | 用户头像 | `GET /api/me` → `avatar` |

**WebSocket 协议**见 [技术方案设计 §3.1](技术方案设计.md)。

## 2. 广场页（pages/explore/explore）

前端 `data.posts[]`：

| 字段 | 类型 | 后端接口 | 后端字段映射 |
|---|---|---|---|
| `id` | number | `GET /api/feed` | `posts.id` |
| `author` | string | — | `users.nickname`（JOIN） |
| `avatar` | string | — | `users.avatar` |
| `time` | string "X小时前" | — | `posts.created_at` → 后端格式化为相对时间 |
| `content` | string | — | `posts.content` |
| `likes` | number | — | `posts.likes_count` |
| `comments[]` | array | — | 内联返回（见下） |
| `comments[].id` | number | — | `post_comments.id` |
| `comments[].author` | string | — | `users.nickname` |
| `comments[].avatar` | string | — | `users.avatar` |
| `comments[].content` | string | — | `post_comments.content` |
| `liked` | bool | — | 当前用户是否在 `post_likes` 中 |
| `showComments` | bool | — | 前端本地状态，不需后端 |

**发帖**：`POST /api/posts`

| 前端参数 | 后端字段 |
|---|---|
| `postContent` | `content` |
| `postImages[]` | `images`（先上传到 OSS 拿 URL） |

**响应格式**（`GET /api/feed`）：

```json
{
  "posts": [
    {
      "id": 12345,
      "author": "陈默",
      "avatar": "https://...",
      "time": "2小时前",
      "content": "...",
      "likes": 124,
      "comments": [
        { "id": 101, "author": "林夕", "avatar": "https://...", "content": "..." }
      ],
      "liked": false
    }
  ],
  "hasMore": true,
  "cursor": "..."
}
```

## 4. 记忆库页（pages/memory/memory）

### 4.1 洞察列表

前端 `data.insights[]`：

| 字段 | 类型 | 后端接口 | 后端字段映射 |
|---|---|---|---|
| `id` | number | `GET /api/memory/insights` | `id` |
| `date` | string "YYYY.MM.DD" | — | `created_at` 格式化 |
| `title` | string | — | AI 生成的 insight 标题 |
| `content` | string | — | AI 生成的 insight 内容 |
| `tag` | string | — | 维度 label 映射为分类名 |
| `tagColor` | string | — | 按分类定死映射（生活→secondary, 情绪→tertiary, 兴趣→primary, 成长→primary） |
| `category` | string | — | `life/emotion/hobby/growth` |

### 4.2 标签/分类

前端 `data.categories[]` 写死为 `全部/生活/情绪/兴趣/成长`。后端 insight 的 `category` 字段必须是这 4 个值之一。

维度 → category 映射：

| 维度 | category |
|---|---|
| D1/D2/D9 | life |
| D5/D7 | emotion |
| D4/D8 | hobby |
| D3/D6/D10 | growth |

## 10. 通用约定

### 错误响应

```json
{ "code": "AUTH_REQUIRED", "message": "请先登录" }
```

### 分页

游标分页，参数 `cursor` + `limit`（默认 20）：

```json
{ "data": [...], "hasMore": true, "cursor": "xxx" }
```

### 时间格式

- 后端存储：ISO 8601（UTC）
- 返回给前端的 `time` 字段：格式化为相对时间（"X分钟前"、"X小时前"、"昨天"、"3天前"、"YYYY.MM.DD"）
- 阈值：< 1h 显示分钟，< 24h 显示小时，< 7d 显示天，≥ 7d 显示日期
