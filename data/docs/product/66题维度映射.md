# 66 题人格维度映射

**上游**：[产品概念文档 §4.3](产品概念文档.md)
**原始数据**：[66题.xlsx](66题.xlsx)
**用途**：把 66 道原始题目归并到可执行的人格维度，供对话编排层做"隐式采样 + 证据抽取 + 置信度更新"。

## 设计约定

- **用户感知**：不是问卷，是和 AI 自然聊天。
- **题目形态**：原始题目是问卷（单选 / 多选 / 7 级量表 / 开放题），不直接问用户；每题改写成 `probe_hints` —— 自然聊天话术。
- **维度模型**：66 题归并为 10 个维度；每个维度有 1–N 题作证据来源。
- **题目复用**：同一道题可以同时供给多个维度，但要标注主从关系。
- **打分方式**：
  - `scale_1_7` 题目：归一化到 0–1
  - `single_choice` / `multiple_choice`：按选项映射到维度分值，在 `questions.extract_rules` 里配置
  - `text`（开放题）：LLM 抽取语义 → 关键词 → 映射维度
- **置信度**：每条证据 `confidence_delta ∈ [0, 0.3]`；维度总置信度累加上限 1.0。

## 采样策略（对话编排用）

1. **首轮开场**：优先覆盖 D6（沟通风格） + D5（社交）+ D4（生活节奏）。这几个维度最自然、最不像测试。
2. **中段**：逐步探 D1 / D3 / D7，这些是匹配核心但用户敏感。
3. **收尾**：如果总时长允许，插入 D8 / D9 的轻松话题；敏感的 D10 留到后续多轮对话再触及。
4. **跨轮复用**：单次会话最多覆盖 5–6 个维度（每维 1–2 条证据即可），剩余维度下一轮补齐。
5. **饱和判断**：所有高权重维度（D1、D2、D3、D6、D7）置信度 ≥ 0.6 → 允许生成人格卡。

## 落库建议

- 维度定义 → `dimensions` 表
- 题目明细 → `questions` 表，`probe_hints` 和 `extract_rules` 存 JSON 字段
- 抽取规则 → 建议用 YAML 集中维护，通过 seed 脚本批量导入。本文档的"题号映射"即为 seed 的权威来源。
