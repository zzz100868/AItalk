# 前后端字段对齐表

**原则**：后端 API 返回的字段名、类型、结构严格贴合前端 `data` 中的字段。前端不改，后端做 DTO 映射。

**数据来源说明**：当前前端使用 `data/mockData.js` 提供初始数据，`stores/userStore.js` 管理用户资料，`utils/common.js` 中的 `storage` 封装 localStorage 读写。

## 1. 首页 / AI 语音通话（pages/index/index）

前端有两种模式：`viewMode = 'landing'`（默认着陆页）和 `viewMode = 'call'`（通话中）。

前端 `data`：

| 字段 | 类型 | 含义 | 后端来源 |
|---|---|---|---|
| `viewMode` | string `'landing'|'call'` | 当前视图模式 | 前端本地状态 |
| `isCalling` | bool | 当前是否通话中 | 前端本地状态 + WS 连接状态 |
| `callDuration` | string "mm:ss" | 通话时长 | 前端本地计时（`callStartedAt` 时间戳） |
| `callDate` | string | 上次通话时间 | `GET /api/voice/sessions?limit=1` → `ended_at` 格式化为 "今天 HH:mm" |
| `isMuted` | bool | 静音 | 前端本地 |
| `isSpeakerOn` | bool | 免提 | 前端本地 |
| `userName` | string | 用户昵称 | `common.loadUserInfo().name` → storage `userProfile.nickName` |
| `aiName` | string | AI 名字 | `mockData.AI_USERS.xiaoya.name` = "小雅" |
| `aiAvatar` | string | AI 头像 URL | `mockData.AI_USERS.xiaoya.avatar` |
| `userAvatar` | string | 用户头像 | `common.loadUserInfo().avatar` → storage `userProfile.avatar` |

**通话结束行为**：`endCall()` → 设置 `getApp().globalData.memoryTargetTab = 'archive'` → `wx.switchTab` 跳转到记忆库页的 archive 子 tab。

**WebSocket 协议**见 [技术方案设计 §4.1](技术方案设计.md)。

| `userAvatar` | string | 当前用户头像 | `common.loadUserInfo().avatar` |
| `matchAvatar` | string | 匹配对象头像 | `POST /api/match/do` → `avatar` |

## 3. 记忆库页（pages/memory/memory）— Tab 1

页面有 3 个子 tab：`activeTab = 'chat' | 'memory' | 'archive'`

### 3.1 通用字段

| 字段 | 类型 | 后端来源 |
|---|---|---|
| `userName` | string | `common.loadUserInfo().name` |
| `userAvatar` | string | `common.loadUserInfo().avatar` |
| `aiAvatar` | string | `mockData.AI_USERS.stitch.avatarSmall`（记忆库用 Stitch AI） |
| `activeTab` | string | 前端本地；可通过 `globalData.memoryTargetTab` 跳转 |
| `tabSliderX` | number | 前端本地（0/100/200 对应 chat/memory/archive） |

### 3.2 Chat 子 tab

| 字段 | 类型 | 后端接口 | 说明 |
|---|---|---|---|
| `messages[]` | array | `GET /api/memory/chat` | 消息列表 |
| `messages[].id` | string/number | — | 消息 ID |
| `messages[].sender` | string `'user'|'ai'` | — | 发送者 |
| `messages[].content` | string | — | 内容 |
| `inputValue` | string | — | 前端本地（输入框） |
| `isSending` | bool | — | 前端本地（发送中状态） |
| `chatDays` | string "12天" | `GET /api/memory/chat` meta | 聊天天数统计 |
| `chatMood` | string "平静" | — | 当前情绪 |
| `chatTopics` | number | — | 话题数 |

**发送消息**：`POST /api/memory/chat` `{ content }` → `{ reply: { id, sender: 'ai', content } }`

当前前端实现：mock 逐字打字效果（从 `MEMORY_REPLIES` 随机取回复）。

### 3.3 Memory 子 tab（洞察列表）

| 字段 | 类型 | 后端接口 | 后端字段映射 |
|---|---|---|---|
| `categories[]` | array | 前端写死 | `[{id:'all',name:'全部'},{id:'life',name:'生活'},{id:'emotion',name:'情绪'},{id:'hobby',name:'兴趣'},{id:'growth',name:'成长'}]` |
| `activeCategory` | string | — | 前端本地筛选状态 |
| `insights[]` | array | `GET /api/memory/insights` | — |
| `insights[].id` | number | — | `id` |
| `insights[].date` | string "YYYY.MM.DD" | — | `created_at` 格式化 |
| `insights[].title` | string | — | AI 生成的 insight 标题 |
| `insights[].content` | string | — | AI 生成的 insight 内容 |
| `insights[].tag` | string | — | 维度 label（"生活"/"情绪"/"兴趣"/"成长"） |
| `insights[].tagColor` | string | — | `secondary/tertiary/primary` |
| `insights[].category` | string | — | `life/emotion/hobby/growth` |
| `filteredInsights[]` | array | — | 前端按 category 过滤后的列表 |

**tagColor 映射规则**：
- `life` → `secondary`
- `emotion` → `tertiary`
- `hobby` → `primary`
- `growth` → `primary`

**本地持久化**：insights 存入 `storage.get('memoryInsights')`，支持编辑和删除。

### 3.4 Archive 子 tab（人格档案）

| 字段 | 类型 | 后端接口 | 说明 |
|---|---|---|---|
| `aboutMe` | string | `GET /api/memory/archive` | AI 生成的人格总结 |
| `personalities[]` | array | — | 性格特征列表 |
| `personalities[].name` | string | — | 如"内向而敏感" |
| `personalities[].desc` | string | — | 详细描述 |
| `traits[]` | array | — | 标签云 |
| `traits[].name` | string | — | 如"深度思考者" |
| `traits[].color` | string | — | `warm/sun/night/mint/bloom/sky` |

**后端响应格式**（`GET /api/memory/archive`）：

```json
{
  "aboutMe": "你是一个在安静中寻找力量的人...",
  "personalities": [
    { "name": "内向而敏感", "desc": "你喜欢独处..." }
```

## 5. 编辑资料页（pages/editProfile/editProfile）

前端提交字段 → `PUT /api/me`：

| 前端字段 | 后端字段 | 备注 |
|---|---|---|
| `avatar` | `avatar` | 先上传 OSS 拿 URL |
| `nickName` | `nickname` | 必填，不能为空 |
| `bio` | `bio` | 可选 |

**注意**：当前 editProfile 页只编辑 avatar/nickName/bio 三个字段。旧文档中的 gender/city/birthYear/orientation 在当前前端无编辑入口。

**数据流**：`userStore.updateProfile(profile)` → 写 localStorage `userProfile` + 更新 `globalData`。

## 10. globalData（app.js）

```js
globalData: {
  userInfo: {
    nickName: '林夕',      // 默认值，启动时从 storage 覆盖
    avatarUrl: '',         // 启动时从 storage 覆盖
    id: 'LX_9527'
  },
  _cache: {               // 内存缓存，避免反复读 storage
    profile: {},          // = wx.getStorageSync('userProfile')
    settings: {}          // = wx.getStorageSync('userSettings')
  },
  memoryTargetTab: null   // 跳转记忆库时指定子 tab ('chat'|'memory'|'archive')
}
```

## 11. 通用约定

### 错误响应

```json
{ "code": "AUTH_REQUIRED", "message": "请先登录" }
```

### 分页

游标分页，参数 `cursor` + `limit`（默认 20）：

```json
{ "data": [...], "hasMore": true, "cursor": "xxx" }
```

### 时间格式

- 后端存储：ISO 8601（UTC）
- 返回给前端的 `time` 字段：格式化为相对时间（"X分钟前"、"X小时前"、"昨天"、"3天前"、"YYYY.MM.DD"）
- 阈值：< 1h 显示分钟，< 24h 显示小时，< 7d 显示天，≥ 7d 显示日期
