# ADR-0002: 真人认证选用腾讯云人脸核身

**状态**：建议，待 PM 确认预算
**当前状态（2026-05 校准）**：技术选型保留，前端入口待建 — 产品概念文档将真人认证列为 MVP Must Have，但当前前端无认证入口（accountSecurity 页无人脸核身调用，pages 中无 facialRecognition 相关代码）。腾讯云人脸核身选型不变，后端鉴权模块开发时需优先接入。
**日期**：2026-04-24
**决策者**：技术负责人

## 背景

产品要求用户必须通过真人认证（人脸识别 / 活体检测）才能参与匹配。需要选择供应商。

## 选项

| 供应商 | 优势 | 劣势 |
|---|---|---|
| **腾讯云人脸核身** | 微信生态原生（`wx.startFacialRecognitionVerify`）；公安一所数据源；小程序 appid 直接绑定 | 价格偏高（约 1 元 / 次） |
| 阿里云人脸认证 | 价格稍低；小程序 SDK 较成熟 | 非微信原生，需嵌入 H5；合规链路多一跳 |
| FaceID（第三方） | 价格低 | 微信审核不确定；合规资质需确认 |

## 决定

**建议选腾讯云人脸核身**。

## 理由

1. **微信原生接口**：`wx.startFacialRecognitionVerify` 体验最流畅，不需要跳转 H5 或 WebView
2. **合规**：公安一所数据源，实名认证法律效力明确
3. **审核**：微信小程序审核对自家服务的接入最宽松
4. **接入成本**：后端只需拿 bizToken 发起 + 查询结果，工程量小

## 后果

- 每次认证约 1 元，需要 PM 确认是否接受（百万用户级 = 百万元认证成本）
- 身份证号不明文存储，只存 hash + verified 标记
- 需要申请腾讯云人脸核身服务 + 配置小程序 appid 绑定
